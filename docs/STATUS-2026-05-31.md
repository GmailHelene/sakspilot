# Sakspilot: Status pr 31. mai 2026

Snapshot av faktisk tilstand i prod. Oppdateres ved hver større leveranse.

## TL;DR

Sakspilot er **funksjonelt komplett for pilotbruk og er nå klar til å distribueres til testbrukere**. Web + API + desktop-agent er live, glemt-passord på epost virker (Brevo), Sentry fanger feil, og desktop-appen har fått en stor opplevelses-oppgradering: **én bryter for auto-spor** (alt åpnet via Sakspilot telles automatisk uten matching-regler), multi-tab snarveier, og egne ikon-opplastinger i launcher.

Mangler kun: Stripe-aktivering (Q1 2027), Bedrock-dependency-install (kun ved behov), og partner-godkjenning hos Tripletex (Fiken er løst via personal-access-token).

## Auto-spor (ny arkitektur)

Implementert 31. mai. Hovedidé: én bryter, ingen regler trengs.

**Komponenter:**
- `apps/desktop/src/settings.js`, `autoTrackOpened`, `activeSakId`, `activeSakTitle` i electron-store
- `apps/desktop/src/poller.js`, `logOpenedExternal()` (eksplisitt session-logging) + `setActiveSakFallback()` (default-attribusjon når matching-regler ikke matcher)
- `apps/desktop/src/main.js`:
  - `setAutoTrack(enabled)`, bryter på/av, starter arbeidsøkt automatisk når på
  - `setActiveSak(sakId, title)`, registrerer aktiv sak
  - `ensureWorkSessionForOpen()`, auto-starter arbeidsøkt når noe åpnes
  - Hooket inn i alle fire `shell:open-*` IPC-handlers
  - Tray-meny øverst: `🎯 Auto-spor PÅ/AV` (toggle) + viser aktiv sak under
  - `agent:status` returnerer nå `autoTrackOpened` + `activeSakTitle` til web
- `apps/web/src/components/DesktopAgentControls.tsx`, pill-bryter i widget med PÅ/AV-badge og "↳ tilordnes <sak>"-indikator
- `apps/web/src/app/saker/[id]/page.tsx`, useEffect pinger `setActiveSak()` når sak åpnes, nullstiller ved navigering

**Brukerflyt:**
1. Bruker slår på "Auto-spor" én gang (tray eller widget)
2. Arbeidsøkt starter automatisk
3. Alt åpnet via Launcher/Mine sites/Mine mapper/.exe logges som session
4. Hvis bruker er på sak-side → tilordnes dit. Ellers `sakId=null` (tilordnes manuelt senere via rapport).
5. Eksisterende matching-regler funker fortsatt og overstyrer fallback.

**Ingen DB-migrasjon**, kun lokal electron-store + utvidet status-payload.
