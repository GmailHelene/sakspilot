# Sakspilot — Status pr 1. juni 2026

Snapshot av faktisk tilstand i prod. Oppdateres ved hver større leveranse.

## TL;DR

Sakspilot er **funksjonelt over enhver pilot-terskel**. Web + API + desktop-agent er live, alle høyt-prioriterte features (auto-spor, AI rate-limit, klistrelapp-reminders, feedback-form) er på plass, OG vi har nå multi-tenancy (team-plan + klient-portal), whitelabel-domener, og cross-platform desktop (Mac/Linux scaffolding klar). Demo-video ligger på forsiden.

Eneste eksterne avhengighet: Tripletex partner-godkjenning (søknad inne).

## Store leveranser siden 31. mai

### Multi-user + multi-tenancy
- **Team-plan med invite-flyt** — owner inviterer member/admin via epost, plan-gating, beskyttet mot siste-owner-degradering
- **Klient-portal med login** — egne brukere for klienter, separat JWT-scope (`scope=client`), kan se sine egne prosjekter + fakturahistorikk på `/portal/*`
- **Custom domener (whitelabel)** — frilanser legger til `klienter.helenetech.no` → klient ser sin egen branding (navn/tagline/farge/logo) i portalen

### Desktop cross-platform
- **Mac + Linux build-pipeline** — `active-win` istedenfor `get-windows`, `--platform=darwin|linux|win32` i build-portable.js, GitHub Actions matrix-workflow over windows/macos/ubuntu
- **/last-ned auto-detects OS** og viser riktig CTA

### Brukervennlighet
- **Pomodoro-timer** i desktop-tray (25/5-økt + notifikasjon)
- **Klistrelapp-reminders** med tidspunkt — in-app toast + native OS-notif (web + desktop)
- **Voice notes på klistrelapper** — MediaRecorder, max 60 sek, lagret som base64
- **Dark mode** — toggle i /innstillinger/utseende
- **Tids-mål per uke/mnd** med widget på /hjem (prorata-target, status-fargekoder)
- **Mobil-kanban** — touch-modal for status-bytte
- **Splash/landing-redesign** med testimonials, sammenligningstabell, FAQ
- **Demo-video** på forsiden (90 sek advokat-flyt, embedded)

### AI + automasjon
- **AI rate-limit per org** — 500/mnd (pilot 5000), token-tracking
- **AI chat-historie per sak** — Claude husker forrige spørsmål innen samme prosjekt
- **AI auto-triage** — foreslår sak for ukategoriserte time-entries
- **Daglig automasjons-trigger** (GitHub Actions, `0 0 * * *` UTC)
- **Onboarding-emails drip** — dag 0/3/7/14 via Brevo

### Eksport + integrasjoner
- **Faktura-PDF** (pdfkit, MVA 25%) — ved siden av Fiken-knappen
- **Rapport-PDF** (per sak/dag/app) — ved siden av CSV
- **iCal-feed** for frister + milepæler — abonner med URL i Google/Apple/Outlook

### Vedlikehold
- **Pilot-feedback-form** (/feedback) med 3 spørsmål + video-call-checkbox
- **Månedlig DB-backup** (GitHub Actions, pg_dump → artifact eller R2)
- **Native always-on-top widget** for desktop (slipper å forsvinne bak snarvei-views)
- **saker/[id]/page.tsx splittet** fra 2049 → 194 linjer (8 section-komponenter)
- **sak → prosjekt UI-rename** — 95 string-bytter i 44 filer. Bruker ser nå "Prosjekter" overalt. Kode + DB beholdt som "sak"/"saker" for bakoverkompatibilitet.

## Tekniske gjenståender / tech-debt

- Service worker disabled i Electron (chunk-hash-fix)
- Render free-tier krever betalt for Shell-access (workaround: lokal db:push)
- Cross-platform build krever GitHub Actions matrix (kan ikke bygges fra én host pga native moduler)
- `release/`-mappe må noen ganger falle tilbake til `release-<timestamp>/` pga antivirus-lås
- iCal-feed cache er per-server-instans (ikke distributed) — OK på Render single-instance
