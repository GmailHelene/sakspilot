## Sammendrag

Sakspilot er et **fullt operativt regnskaps- og prosjektverktøy** for selvstendig
næringsdrivende i Norge. På én dag (3. juni) ble en bred runde av feature-arbeid
gjennomført som flytter verktøyet fra "fungerer for pilot" til
"konkurrent på beste norske SaaS-er for små bedrifter":

- **Hub-funksjonene importert som ekte features**: Forespørsler, Fakturaer,
  Regnskap, MVA-rapport, Statistikk
- **Send faktura som epost direkte** (Brevo HTTP API, vedlegg PDF)
- **Faktura-purring** med 3-trinns eskalering
- **Bank-CSV-import** av utgifter (DNB / Sparebank1 / Nordea / generic)
- **Kvittering-upload** på utgifter (lov-krav 5 års oppbevaring)
- **Klient-portal utvidet** med fakturaer-fane
- **Søk overalt** med debounce
- **Inline-rediger** på forespørsel-kanban
- **Sidebar-collapse** ("Mer..." for ekstra-fane)
- **Mobil-fakturaer** som kort-stack
- **Native Windows-toast** for nye leads (med lyd) + Mac/Linux .dmg/.tar.gz
- **Auto-badges** på snarvei-tiles (Gmail "Inbox (3)" detekteres automatisk)
- **Manuelle badges** høyreklikk → sett antall
- **PDF-rate-limit** + diverse sikkerhetsforbedringer

## Bruks-guide for sluttbrukere

### "Send min første faktura via Sakspilot"

1. Lag prosjekt + klient (eller bruk eksisterende)
2. `/fakturaer` → "+ Ny faktura"
3. Velg sak (eller skriv kundenavn fritt), legg til linjer, sett forfall
4. "Opprett som utkast"
5. Åpne den nye fakturaen → "Send på epost"
6. Mottaker pre-fylles fra klientens kontakt-epost
7. "Send nå" → PDF som vedlegg, sendt via Brevo

### "Importer fakturaer fra bank-CSV"

1. Eksporter CSV fra nettbanken (DNB/Sparebank1/Nordea støttet)
2. `/regnskap` → "Bank-CSV"
3. Velg fil → preview viser autodetektert format + alle utgifter
4. Velg/fjern rader, sett kategori per rad
5. "Importer N utgifter"
6. Duplikat-sikker (externalId per rad)

### "Send purring til kunde"

1. `/fakturaer` → åpne en forfalt+ubetalt faktura
2. Oransje "Send purring"-knapp vises
3. Klikk → bekrefte → epost med PDF "PURRING"-stempel sendes
4. reminderCount øker, neste klikk gir andre påminnelse
5. Tredje gang = "Siste purring før inkasso"-tekst

### "Last opp kvittering på utgift"

1. `/regnskap` → "Ny utgift" (eller eksisterende rad)
2. Klikk "Velg fil" / "Legg ved" → JPG/PNG/PDF (max 5MB)
3. Thumbnail vises i Kvittering-kolonnen
4. Klikk thumbnail → full preview i ny tab
5. Lov-krav 5 års oppbevaring — lagres i DB

### "Klient ser sine fakturaer"

1. Som frilanser: `/klienter/:id` → "Invitér til portal"-knapp
2. Klient får epost → setter passord
3. Klient logger inn på `https://sakspilot.no/portal/login`
4. Ser sine prosjekter på `/portal` + fakturaer på `/portal/fakturaer`
5. Kan IKKE se utkast eller annulerte fakturaer, kun "exported"

## Beslutninger og avveininger

- **base64 i DB for kvittering** istedenfor S3/Vercel Blob: Enkelhet > skalering for nå
  (ved 1000 kvitteringer = ~6.5GB i DB; OK på Neon Pro). Migrer hvis nødvendig senere
- **Brevo HTTPS API** istedenfor SMTP: Render Free-tier blokkerer utgående SMTP-porter
- **Klient-side CSV-parsing** istedenfor server: Personvern (transaksjonsdata
  forlater aldri brukerens nettleser før bekreftet preview)
- **Manuell faktura-PDF kode duplikat** i `invoicePdf.ts` og `invoices.ts/send-email`:
  TODO refaktor til shared lib når en av dem endrer seg
- **Inline-rediger kun på foresporsler-kanban** for nå: Forventet bruksmønster
  for "rask rediger". Andre sider kan utvides etterhvert
- **NavBadge fjernet grå "alle lest"-modus**: Forvirrende på Prosjekter med 8 forfalte
  saker — det er ikke nytt, brukeren har bare ikke fikset
- **NotifPoller exp backoff** ved API-feil: 60s → 120 → 240 → 300 max,
  reset ved første vellykkede tick
